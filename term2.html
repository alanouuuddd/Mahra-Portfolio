Term 2

Systems & Security Foundations

Diving deeper into how computer systems work, how passwords are protected, and how laws keep us safe in the digital world.

In Term 2, I explored the technical side of cybersecurity — how systems are layered for protection, how Python is used to secure passwords through hashing, and the real dangers of the dark web. I also studied cybersecurity laws in the UAE and the growing risks of connected IoT devices.

Topic 01

System Layers – How Computers Are Structured & Protected

Computer systems are organized in layers, each requiring its own security measures. Understanding the layers helps us identify where attacks can happen.

👤User Layer — Passwords, 2FA, User Permissions
💻Application Layer — Secure Software, Input Validation
🔄OS Layer — System Updates, Access Control
🌐Network Layer — Firewalls, Encryption, VPNs
🖥Hardware Layer — Physical Security, Secure Boot

Topic 02

Password Hashing Using Python

Storing passwords in plain text is dangerous. Hashing converts a password into a fixed-length string that cannot be reversed. We used Python's hashlib library to demonstrate this.

# Password Hashing with Python - hashlib import hashlib # The user's password password = "SecurePass123" # Encode and hash using SHA-256 hashed = hashlib.sha256(password.encode()).hexdigest() print("Original: ", password) print("Hashed: ", hashed) # Output: # Hashed: a665a45920422f9d417e4867efdc4fb8...
Key Concept: Even if attackers steal the database, they only see hashed values — not real passwords. Salting adds extra randomness to prevent rainbow table attacks.

Topic 03

Dark Web & Cybersecurity Risks

The internet has three layers. Hover over each layer of the iceberg below to explore what it contains and the risks it carries.

Risks of the Dark Web: Stolen credentials for sale, illegal marketplaces, malware downloads, and cybercriminal networks.
🌐 Surface Web ~4% of internet
Indexed by search engines (Google, Bing). Includes public websites, social media, news, and videos. Accessible to everyone.
≈ W A T E R L I N E ≈
🔒 Deep Web ~90% of internet
Not indexed by search engines. Includes online banking portals, medical records, email inboxes, and private databases. Requires login credentials to access.
🌒 Dark Web ~6% of internet
Requires special software (Tor browser) to access. Contains illegal markets, stolen data, weapons, and criminal networks. Highly dangerous.
Tor Browser Anonymous Networks Data Breaches

Topic 04

Cybersecurity Laws & Digital Safety Rules

The UAE has strict laws to protect people online. The UAE Cybercrime Law (Federal Decree-Law No. 34 of 2021) criminalizes hacking, identity theft, spreading fake news, and unauthorized data access.

Penalties include: Fines up to AED 3,000,000 and imprisonment for serious cybercrimes including unauthorized access and data theft.
UAE Cybercrime Law GDPR Awareness Digital Rights
🔌
Topic 05

Internet of Things (IoT) & Cybersecurity Risks

IoT refers to everyday devices connected to the internet — smart TVs, refrigerators, cameras, thermostats, and more. Each connected device is a potential entry point for attackers.

Main Risks: Default passwords left unchanged, outdated firmware, no encryption, and devices that cannot be easily updated or patched.
Smart Home Risks Botnet Attacks Firmware Updates

💡 Term 2 Key Takeaway

Security is not just one thing — it is many layers working together. From hashing passwords in Python to understanding the dark web and UAE laws, I learned that both technical skills and legal knowledge are needed to stay safe and responsible in the digital world. The growth of IoT also shows that cybersecurity must evolve with the technology we use every day.
